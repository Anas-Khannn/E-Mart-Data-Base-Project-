Terms & Conditions

1. DBMS System Access and Integrity

This system is authorized strictly for E-Mart employees. Any attempt to modify hardcoded indices, execute unauthorized scripts, or manipulate pricing data grids in any terminal/console viewport constitutes a breach of the university dbms protocol guidelines.

2. POS and Stock Verification Protocol

All cashiers, manager associates, and checkout agents are required to thoroughly audit item counts. The hardcoded product dataset matches real inventory registers, and any discrepancies should be logged on the dashboard control feed.

3. Role and Access Control Restrictions

System privileges (Admin, Manager, Cashier, Staff) strictly regulate POS checkout, invoice printing, inventory updates, and visual chart reporting. Staff are forbidden from sharing login keys or operational credentials.

4. Static Data Simulation

Please note: As a static university DBMS submission layout, all database structures, customer entries, transactions, and analytics dashboards represent simulated client data blocks.
